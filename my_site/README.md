# Module 01: Django Installation and Project Setup

This folder contains the initial scaffolding of our Django web application, representing the completion of the first core developmental milestone. The goal of this module was to establish a functional development server and explore the default operational files generated by the framework.

---

## Generated Architecture Overview

When the initialization command was executed, Django automatically structured the core engine components inside this directory:

*   **`manage.py`**: The command-line utility used to interact with this project (e.g., starting the server, handling migrations).
*   **`my_site/settings.py`**: The central configuration brain of the application, managing database connections, registered apps, and static assets.
*   **`my_site/urls.py`**: The URL routing directory that maps web requests to specific views.
*   **`my_site/wsgi.py` & `asgi.py`**: Deployment hooks for web servers to serve the application in production.
